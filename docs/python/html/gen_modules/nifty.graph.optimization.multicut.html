class nifty.graph.optimization.multicut.CgcMulticutObjectiveEdgeContractionGraphUndirectedGraph¶

Multicut objective solver Cgc.

Cut glue & and cut (CGC) [BKK+14] works by applying a series Of two-colorings / min-cuts to optimize the multicut problem. The algorithm works as illustrated in the figure below.

Left: The graph is recursively partitioned by applying a min-cut on each connected component. Right: Each adjacent pair of connected components is merged and optimized again by searching for a new min-cut / 2-coloring. This is repeated until nothing improves anymore.

See also

Objective Class: MulticutObjectiveEdgeContractionGraphUndirectedGraph

Graph Class: nifty.graph.EdgeContractionGraphUndirectedGraph

Solver Base Class: MulticutBaseMulticutObjectiveEdgeContractionGraphUndirectedGraph

Solver Factory Class: CgcFactoryMulticutObjectiveEdgeContractionGraphUndirectedGraph

Cite: [BKK+14]

Note

This solver should be warm started,otherwise the glue phase is very slow.Using greedyAdditiveFactory() to create a solver for warm starting is suggested.

class nifty.graph.optimization.multicut.MulticutObjectiveEdgeContractionGraphUndirectedGraph¶

Multicut objective for a graph of type nifty.graph.EdgeContractionGraphUndirectedGraph

The multicut objective function is given by:

\[ \begin{align}\begin{aligned}E(y) = \sum_{e \in E} w_e \cdot y_e\\st. y \in MulticutPolytop_{G}\end{aligned}\end{align} \]

This energy function can be used to find the optimal multicut:

\[ \begin{align}\begin{aligned}y^* = argmin_{y} \sum_{e \in E} w_e \cdot y_e\\st. y \in MulticutPolytop_{G}\end{aligned}\end{align} \]

  • sizeRegularizer – If doBetterCutPhase is True, we use a agglomeration to create a set of clusters. If this number is larger as zero, the clusters have about equal size (default: {1.0})
